1"""Correct race-elapsed timing under clock anomalies. 

2 

3Race timing must survive two clock hazards that can occur *during* a race: 

4 

5* an **NTP or manual step** of the wall clock, forward or backward; and 

6* the machine going to **sleep** — endurance races run for hours or days, and a 

7 laptop lid can close by accident; real race time keeps elapsing while asleep. 

8 

9``time.monotonic`` is immune to NTP steps but on Linux/macOS it *pauses* during 

10system suspend, so on its own it under-counts a sleep. The wall clock counts sleep 

11but is NTP-corruptible. This module yields an elapsed time correct under both, by 

12choosing the cleanest correct source per platform: 

13 

14* **Linux** — ``CLOCK_BOOTTIME``: a single native clock that is monotonic, 

15 NTP-immune, *and* counts suspend. Exact; no heuristic. 

16* **Windows** — ``time.monotonic`` (QueryPerformanceCounter) counts suspend, so it 

17 is exact for both hazards too. 

18* **Otherwise** (macOS, or any platform whose monotonic clock turns out to freeze 

19 on suspend) — a dual-clock reconstruction from wall + monotonic (:class:`DualRaceClock`) 

20 that recovers a sleep gap from the wall clock while ignoring NTP steps via the 

21 monotonic one. 

22 

23All arithmetic is in **integer nanoseconds**. Python ``int`` is arbitrary 

24precision, so ``elapsed_ns`` cannot overflow even for a days-long race, and there 

25is no floating-point accumulation drift; callers convert the small per-lap delta 

26to seconds only at the end. 

27""" 

28import sys 

29import time 

30 

31# Wall-vs-monotonic divergence beyond this (dual-clock fallback only) is treated as 

32# real time the monotonic clock missed — a system sleep. It sits far above awake 

33# jitter (sub-millisecond) and far below any real gap. A forward NTP step is 

34# indistinguishable from a sleep here and is treated as one: the fail-safe 

35# direction (never under-count race time). 

36_GAP_NS = 500_000_000 # 0.5 s 

37 

38 

39class RaceClock: 

40 """Elapsed race time in integer nanoseconds from a single sleep-inclusive, 

41 NTP-immune monotonic clock (Linux ``CLOCK_BOOTTIME`` / Windows monotonic). 

42 

43 ``clock_ns`` (a callable returning int ns) is injectable for tests. ``read_ns`` 

44 is a pure read — ``clock_ns() - origin`` — so it can be called at any rate. 

45 """ 

46 

47 def __init__(self, clock_ns): 

48 self._clock_ns = clock_ns 

49 self._origin = None 

50 

51 def start(self, elapsed_ns=0): 

52 """Pin the origin. Pass ``elapsed_ns > 0`` to resume a race already that 

53 many nanoseconds in (e.g. bridging downtime after a restart).""" 

54 self._origin = self._clock_ns() - int(elapsed_ns) 

55 

56 def read_ns(self): 

57 return 0 if self._origin is None else self._clock_ns() - self._origin

60class DualRaceClock: 

61 """Fallback for platforms lacking a sleep-inclusive monotonic clock: 

62 reconstruct elapsed from a wall clock plus a (sleep-pausing) monotonic clock. 

63 

64 Per sampled interval (``read_ns`` is called at each lap, and may be polled for a 

65 live display), advance by the **monotonic** delta — which never jumps on an NTP 

66 step — *except* when the wall clock ran ahead of it by more than ``_GAP_NS``, 

67 which means real time passed while monotonic was frozen (a sleep): then advance 

68 by the **wall** delta to recover it. A backward wall step (NTP) shows up as the 

69 wall running behind monotonic and is ignored. Elapsed never decreases. 

70 

71 Correct regardless of whether the platform's monotonic already counts sleep: if 

72 it does, ``dw ≈ dm`` so nothing is double-counted; if it doesn't, the gap is 

73 recovered. 

74 """ 

75 

76 def __init__(self, wall_ns, mono_ns): 

77 self._wall_ns = wall_ns 

78 self._mono_ns = mono_ns 

79 self._elapsed = 0 

80 self._last_wall = None 

81 self._last_mono = None 

82

84 self._elapsed = int(elapsed_ns) 

85 self._last_wall = self._wall_ns() 

86 self._last_mono = self._mono_ns()
